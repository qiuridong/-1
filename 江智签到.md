# 江智签到进度同步

本文档用于同步 `E:\江智签到` 当前脚本状态、常用命令和 GitHub 仓库操作方式，避免后续会话重复分析。

## 1. 当前结论

- 当前真实可用链路不是旧的 `/portal-api/app/index/login` 直登。
- 根据 HAR 和实测，平台现在采用统一认证登录，最终仍通过：
  - `app_user_id -> /portal-api/app/index/checkAppUserIdNew`
  - 换取实际用于签到的 Bearer token
- 脚本已经改成优先复用缓存 token；token 失效时，再重新走绑定和换 token。
- 绑定流程已经小白化：
  1. 先填写签到地址
  2. 自动解析并保存 GCJ-02 经纬度
  3. 填写默认图、早图、晚图
  4. 最后打开浏览器完成统一认证和验证码

## 2. 已实现能力

- 支持 `--bind-account` 浏览器授权绑定账号
- Playwright 不可用时，自动回退 Selenium + 系统 Edge/Chrome
- 自动保存地址、经纬度、默认图、早图、晚图
- 自动保存 `app_user_id / user_id / autonomy_id / token`
- 上传图片后轮询图片 URL，确认可访问再提交签到
- 写接口返回 `401 / 登录信息已过期` 时，自动重新换 token 并重试一次
- 多账号支持
- 晚上补签规则：
  - 如果当前是晚上，并且当天只有 `1` 条签到记录，则允许补签到第 `2` 次
  - 防止首次启用脚本时已经到晚上，只签到过一次却被误判为“晚上已签到”
- 常驻晚间双签规则：
  - 常驻模式触发晚上任务时，如果当天记录为 `0` 条，会先用早图补一次，再等待 `10` 秒用晚图签到一次
  - 可用 `--run-scheduled-slot-now evening` 立即执行同款逻辑，方便当天测试
- 已修复账号串号问题：
  - 重新绑定账号时会清空旧 `token / user_id / nick_name / autonomy_id`
  - 每次拿到 token 后都会重新校验当前账号的实习计划，避免新账号 token 搭配旧账号 `autonomy_id`

## 5. 常用运行命令

### 5.1 Windows EXE 版

当前 `win` 分支提供打包好的 Windows 可执行文件：

- 下载地址：[jiangzhi-checkin.exe](https://github.com/qiuridong/-1/raw/win/dist/jiangzhi-checkin.exe)
- 配置模板下载：[checkin_config.example.json](https://raw.githubusercontent.com/qiuridong/-1/win/checkin_config.example.json)
- 本地构建产物：[jiangzhi-checkin.exe](E:\江智签到\dist\jiangzhi-checkin.exe)

EXE 版不需要安装 Python，也不需要执行 `python -m playwright install`。它默认使用系统 Edge/Chrome 完成浏览器授权。

当前版本已支持高德地址解析，仓库里只提供空配置项：

```json
{
  "amap_key": ""
}
```

说明：

- GitHub 上只保留空的 `amap_key`，不会提交真实 Key
- 你需要在自己机器上的 `checkin_config.json` 里填写真实 `amap_key`
- 如果 `amap_key` 为空，脚本会回退到旧的 Nominatim 地址解析；该服务在国内服务器上通常不稳定
- 部署 EXE 时，建议把 `jiangzhi-checkin.exe`、`checkin_config.json`、`token.json` 放在同一目录
下面示例假设已经进入 EXE 所在目录：

```powershell
cd /d E:\江智签到\dist
```

如果还没有 `checkin_config.json`，先任选一种方式：

```powershell
Copy-Item .\checkin_config.example.json .\checkin_config.json
```

或：

```powershell
.\jiangzhi-checkin.exe --init-config
```

建议至少先填写这些字段：

- `amap_key`
- `clock_address`
- `proof_image_path`
- `proof_images.morning`
- `proof_images.evening`
首次绑定或重新绑定：

```powershell
.\jiangzhi-checkin.exe --config ..\checkin_config.json --bind-account
```

只测试链路，不真正提交：

```powershell
.\jiangzhi-checkin.exe --config ..\checkin_config.json --once --dry-run
```

真实签到一次：

```powershell
.\jiangzhi-checkin.exe --config ..\checkin_config.json --once
```

立即执行常驻模式同款晚上任务：

```powershell
.\jiangzhi-checkin.exe --config ..\checkin_config.json --run-scheduled-slot-now evening
```

启动常驻定时模式：

```powershell
.\jiangzhi-checkin.exe --config ..\checkin_config.json
```
